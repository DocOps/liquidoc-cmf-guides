Why LiquiDoc?

Docs + Code, sitting in a tree

The “right way” to document software has always been considerd the “hard way”. LiquiDoc is building a bridge across that gap, marrying product docs and product code like never before.

Open as in Free

LiquiDoc and its dependencies are all permissively licensed for commercial use and modification. You can customize it to your needs and maybe contribute your changes back.

Order out of Chaos

LiquiDoc draws structured matter from disparate content and data sources such as YAML, JSON, and AsciiDoc, plus real product code samples straight from the source repo, massaging and compiling everything into rich document artifacts ready for integraton and deployment.

Slash those Tags!

LiquiDoc relies on lightweight markup formats AsciiDoc, YAML, and Liquid to store and structure content and data. Escape the constraints and proprietary tooling of XML; live tag-free with AsciiDoc

“Small Data” FTW

YAML provides an excellent, human-readable, Git-friendly data-storage format that's just the right scale for product metadata and technical references. Flat files and lightweight markup mean accessibility.

Version control

When the docs source is shacking up with the product codebase, docs can stay in tune with the user interfaces they describe, conforming to versioned release branches and environment tags.

Frequently Asked Questions


What platforms does LiquiDoc run on?

LiquiDoc is a Ruby gem, so you’ll need the Ruby runtime environment. If you're a Mac or Linux user, you're probably all set. Windows users may have some preliminary installation steps if they have never installer Ruby. If you're already using Asciidoctor or Jekyll, you're all set.

What are the core technologies behind LiquiDoc

LiquiDoc was designed to enhance existing open-source tools like Asciidoctor (AsciiDoc rendering suite), Jekyll (static site generator), and templating languages such as Liquid.
